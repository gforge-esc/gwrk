# Verification Gate Template

> Gate files are executable shell scripts that assert task acceptance criteria.
> They are committed BEFORE implementation and run by `/implement` and `/review-code`.

## Generation Rules

1. One gate file per beads task: `{feature_dir}/gates/{task_id}-gate.sh`
2. Gates are generated during Step 7a of `/plan-to-beads`
3. Extract the `## Verification` section from each task description into a standalone script
4. Gates MUST be executable (`chmod +x`) and exit 0 on PASS, non-zero on FAIL
5. Gates are committed alongside beads import scripts: `git commit -m "gate: red gates for Phase N"`

## Gate File Format

```bash
#!/bin/bash
# Gate: T0XX — {task title}
# Spec: FR-### / TR-###
# Generated by: /plan-to-beads
# Status: RED (pre-implementation)
set -euo pipefail

GATE_NAME="T0XX"

echo "=== GATE: $GATE_NAME ==="

# --- Assertion 1: {what it checks} ---
# {exact command with grep/diff/exit-code assertion}
# Example:
# pnpm exec tsc --noEmit 2>&1 || { echo "FAIL: TypeScript compilation"; exit 1; }

# --- Assertion 2: {what it checks} ---
# Example:
# test -f packages/config/src/index.ts || { echo "FAIL: config entry missing"; exit 1; }

echo "PASS: $GATE_NAME"
exit 0
```

## Assertion Patterns

| Check Type | Pattern |
|---|---|
| File exists | `test -f {path} \|\| { echo "FAIL: {path} missing"; exit 1; }` |
| TypeScript compiles | `pnpm exec tsc --noEmit 2>&1 \|\| { echo "FAIL: tsc"; exit 1; }` |
| Rust compiles | `cargo check --manifest-path {path} 2>&1 \|\| { echo "FAIL: cargo check"; exit 1; }` |
| Zod schema throws on bad input | `node -e "require('{module}').parse({})" 2>&1 \| grep -q "ZodError" \|\| { echo "FAIL: no ZodError"; exit 1; }` |
| Docker container runs | `docker compose ps --format json \| jq -e '.[] \| select(.Name==\"{svc}\")' \|\| { echo "FAIL: {svc} not running"; exit 1; }` |
| Export exists | `node -e "const m = require('{pkg}'); if (!m.{export}) process.exit(1)" \|\| { echo "FAIL: {export} not exported"; exit 1; }` |
| Unit test passes | `pnpm test -- --filter {file} 2>&1 \|\| { echo "FAIL: unit test"; exit 1; }` |
| Cargo test passes | `cargo test --manifest-path {path} {test_name} 2>&1 \|\| { echo "FAIL: cargo test"; exit 1; }` |

## Anti-Patterns

- ❌ Gate that always passes (`exit 0` with no assertions)
- ❌ Gate that checks only "file exists" when the task is about behavior
- ❌ Gate with prose comments instead of executable assertions
- ❌ Gate that depends on runtime state not available pre-implementation (it's OK — gates start RED)
- ❌ Assertions using `curl` against services not yet running (use `pnpm exec` or `node -e` for static checks)

## Lifecycle

```
/plan-to-beads → generates gates/*.sh (RED — they will fail)
                → commits gates alongside beads scripts
/implement     → for each task, runs gates/{task_id}-gate.sh after coding
               → task is NOT closed unless gate exits 0
/review-code   → runs ALL gates/*.sh in batch as objective pass/fail
```

## Runner Script

`/plan-to-beads` also generates `{feature_dir}/gates/run-all-gates.sh`:

```bash
#!/bin/bash
set -euo pipefail
PASS=0; FAIL=0; TOTAL=0
for gate in "$(dirname "$0")"/T*-gate.sh; do
  TOTAL=$((TOTAL + 1))
  if bash "$gate" 2>&1; then
    PASS=$((PASS + 1))
  else
    FAIL=$((FAIL + 1))
  fi
done
echo "=== GATES: $PASS/$TOTAL pass, $FAIL fail ==="
[ "$FAIL" -eq 0 ] || exit 1
```

# gwrk: The Principal Engineer's Operating System

**gwrk** is a CLI-driven operating system for Principal Engineers. It provides hierarchical command routing, flat-file task tracking with Hard Gate enforcement, agent dispatch wrappers, a SQLite execution ledger, and shell-script orchestration passthroughs.

## Repository Audit & Organization

This repository is strictly organized to enforce separation of concerns between raw implementation, task state, agent governance, and orchestration paths.

### Core Directories
- **`src/`**
  - The TypeScript source code for the `gwrk` CLI binary and its corresponding server instances. No `.js` files are permitted here (TypeScript is the sole source of truth).
- **`specs/`**
  - Feature specifications (`spec.md`), technical plans (`plan.md`), and their isolated task state ledgers.
  - `.gwrk/tasks.json` lives within these feature directories to maintain git-branch isolation for work-in-progress state.
- **`docs/`**
  - High-level architectural documents, PRDs, and the foundational `FOXTROT-CHARLIE.md` taxonomy that governs feature pillars.

### Orchestration & Governance
- **`scripts/dev/`**
  - The deterministic Bash orchestration scripts (`agent-run.sh`, `define-until-solid.sh`, `work-until-done.sh`) that the CLI wraps. *These shell scripts are the actual product orchestrators.*
- **`.agents/`**
  - Governance protocols, agent rules (e.g., `operating-model.md`), and Prompt/Workflow context passed directly to LLMs during execution.
- **`.specify/`**
  - Layout templates used by the `/specify` workflow to scaffold blank or incoming specifications.

### Operations & Infrastructure
- **`tests/`**
  - Broad integration and end-to-end testing definitions.
- **`dist/`**
  - The compiled CommonJS/ESM outputs generated by `tsc` from `src/`.
- **`coverage/`**
  - Output directories for `vitest` code coverage reports.
- **`.gwrk/` (Root)**
  - Local global state (if any) that isn't feature-branch scoped, though execution ledger runs primarily go to `~/.gwrk/gwrk.db`.

### Configuration Files
- **`Makefile`**
  - The primary entry point for developer workflows (e.g., `make test`, `make build`, `make clean`).
- **`package.json` / `pnpm-lock.yaml` / `pnpm-workspace.yaml`**
  - Dependency management via `pnpm`.
- **`tsconfig.json`**
  - Strict TypeScript compiler configurations for the `src/` directory.
- **`biome.json`**
  - The Biome linter and formatter configuration enforcing the code style.
- **`vitest.config.ts`**
  - Test runner configurations for the `vitest` suite.
- **`.gwrkrc.json`**
  - The fail-fast project configuration evaluated and validated strictly by `zod` at runtime.
- **`Dockerfile.sandbox`**
  - Container definition for isolated execution environments during tests or agent runs.

# Getting help with gwrk

gwrk is in active alpha 🦩 — thanks for being here early. Here's the fastest way to get what you need
and to make sure your input lands where a maintainer will see it.

## Where do I put this?

| I want to… | Go here |
|---|---|
| **Report something broken** (reproducible bug) | [New issue → Bug report](https://github.com/gforge-esc/gwrk/issues/new?template=bug_report.yml) |
| **Request a feature** (concrete, actionable) | [New issue → Feature request](https://github.com/gforge-esc/gwrk/issues/new?template=feature_request.yml) |
| **Propose a plugin** (agent / workflow / skill) | [New issue → Plugin request](https://github.com/gforge-esc/gwrk/issues/new?template=plugin_request.yml) |
| **Share what worked / what was rough** (open-ended) | [Discussions → 🧪 Alpha Feedback](https://github.com/gforge-esc/gwrk/discussions/categories/alpha-feedback) |
| **Suggest a direction / idea** (what you'd like to see) | [Discussions → 💡 Ideas](https://github.com/gforge-esc/gwrk/discussions/categories/ideas) |
| **Show what you built with gwrk** | [Discussions → 🙌 Show & Tell](https://github.com/gforge-esc/gwrk/discussions/categories/show-and-tell) |
| **Ask a how-do-I question** | [Discussions → 🙏 Q&A](https://github.com/gforge-esc/gwrk/discussions/categories/q-a) |
| **Report a security vulnerability** | **Do not open a public issue** — follow [SECURITY.md](SECURITY.md) |

## Rule of thumb

- **Issues** are for *actionable work* — a specific bug, or a concrete change with a clear "done."
- **Discussions** are for *everything open-ended* — experience, ideas, questions, and show-and-tell.
- When in doubt, start a **Discussion**. A maintainer can convert it into an issue if it's actionable.

## Before you post

- Skim the [docs](docs/) and search existing
  [issues](https://github.com/gforge-esc/gwrk/issues) and
  [discussions](https://github.com/gforge-esc/gwrk/discussions) — your answer or a +1 target may
  already exist.
- Include your `gwrk --version`, OS, Node version, and which agent CLI you're using (claude, gemini,
  codex, agy).

## Roadmap & direction

Check the public [roadmap](https://github.com/orgs/gforge-esc/projects/5) before requesting something —
it shows what's already **Considering / Planned / In progress**. gwrk's roadmap is a *trust artifact*:
directional and outcome-led, not a dated commitment (see
[Foxtrot Charlie](docs/product/FOXTROT-CHARLIE.md)).

## Response expectations

gwrk is maintained by a small team during alpha. We read everything; we can't promise fast turnaround.
Clear, reproducible bug reports and outcome-framed ideas get actioned fastest.

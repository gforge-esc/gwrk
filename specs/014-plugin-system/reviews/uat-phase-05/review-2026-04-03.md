## Detailed Findings

### 1. Critical Regression: Workflow Resolution (US-011)
Executing `gwrk define spec` or `gwrk specify` results in an `ENOENT` error.
- **Reproducer**: `gwrk init && gwrk define spec my-feature "test"`
- **Actual Output**: `✗ [BLOCKED] ENOENT: no such file or directory, open '/path/to/project/gwrk-specify'`
- **Root Cause**: `src/utils/agent.ts` > `dispatchAgent()` attempts to read the workflow name as a local file path. It must be updated to skip `fs.readFileSync` when a plugin-based workflow is being executed.

### 2. Visibility Issue: Local Overrides (US-015)
While the runtime may resolve local overrides, the CLI visibility is broken.
- **Reproducer**: Create `.gwrk/plugins/workflows/gwrk-specify/manifest.yaml` with an override version.
- **Actual Output**: `gwrk plugin list --project` still shows the built-in version (v0.1.0).
- **Remediation**: `PluginLoader.listPlugins()` needs to include the project-local `.gwrk/plugins` directory in its scan.

# /gwrk-research-ontology

**Persona**: Product Manager / Domain Expert
**Pillar**: Definition (Ontology)

## Purpose
Define and refine the project's domain ontology, identifying core entities, their properties, and relationships.

## Scope Constraints
- MUST focus on domain language and conceptual clarity.
- MUST align with existing grounding documents.
- MUST NOT speculate on implementation details.

## Algorithm
1. Read the research brief.
2. Analyze existing domain documentation.
3. Map entities, relationships, and invariants.
4. Synthesize findings into an ontology research report.
5. Generate `WRITE_FILE` intent for `draft.md` in the initiative directory.
